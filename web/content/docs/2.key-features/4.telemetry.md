## Summary

Zeeq emits and surfaces extensive telemetry to let teams understand how agents are using the knowledge library and code reviews and where to invest time in having senior engineers manage context.

## Dashboard

The main dashboard provides insights into tool usage, agent platforms, as well as activation rate by user.

![telemetry-dash](/telemetry-dash.png)

## Knowledge library telemetry

Knowledge library telemetry surfaces the high-leverage context that is driving agent output across the organization:

![telemetry-kb](/telemetry-kb.png)

## Contextual telemetry

Telemetry is surfaced contextually in code reviews both in the GitHub PR:

![gh-review-metrics](/gh-review-metrics.png)

As well as in each code review record:

![code-review-telemetry](/code-review-telemetry.png)

This insight provides visibility into how the code review agents are using the knowledge library and world model to produce its reasoning and feedback.

## Session telemetry (2026 Q3)

Session level telemetry allows managers and vertical owners to see performance across the team to understand token usage patterns, developer prompting patterns, and additional insights at the session level across all harnesses and models.
